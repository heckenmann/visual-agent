# UC-0000020: Execute Tool Call

## Goal

Execute a model-requested tool safely through the application tool registry and return a structured result.

## Primary Actor

LLM provider.

## Preconditions

- The active request context includes enabled tools.
- The provider emits a valid tool call.

## Main Flow

1. The provider receives a model tool-call request.
2. The tool registry maps provider-safe function names to canonical tool IDs.
3. The selected tool receives JSON input and request metadata.
4. The tool executes with timeout handling.
5. Tool start and finish events are emitted.
6. The structured result is returned to the provider flow.

## Result

Tool behavior is centralized, auditable, and visible in conversation history.

## Tool Calls

- Any request-enabled `VisualAgentTool` can be called through the provider-safe function name generated by `ToolRegistry`.

## Code Entry Points

- `de.heckenmann.visualagent.agent.tools.ToolRegistry`
- `de.heckenmann.visualagent.agent.tools.VisualAgentTool`
- `de.heckenmann.visualagent.agent.tools.ToolEventBus`

## Acceptance Criteria

- Only request-enabled tools are callable.
- Tool events are persisted and rendered.
- Unknown tool names get a controlled failure or recovery response.
